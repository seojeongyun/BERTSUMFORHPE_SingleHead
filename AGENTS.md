# 작업 정책

## 원본 파일 보호

- 원본 파일은 절대 수정하지 않는다.
- 모든 원본 파일은 **읽기 전용(Read-only)** 으로 취급한다.
- 분석 및 실행은 수행할 수 있으나, 원본 데이터, 소스코드, 설정 파일, 체크포인트는 어떠한 경우에도 변경하지 않는다.
- 원본 파일을 덮어쓰거나, 삭제하거나, 이동하거나, 이름을 변경하는 작업은 절대 수행하지 않는다.

## 결과물 생성

- 새롭게 작성하는 코드, 보고서, 로그, 이미지, CSV, JSON 등 모든 결과물은 다음 경로에만 생성한다.

```
/home/jysuh/PycharmProjects/Codex_report
```

- 결과물은 반드시 `/home/jysuh/PycharmProjects/Codex_report` 하위에 **새로운 폴더를 생성한 후** 저장한다.
- 새로 생성하는 폴더명은 **입력된 프롬프트의 주제를 기반으로 명명**한다.
- 프롬프트를 실행할 때마다 기존 폴더를 재사용하지 말고 **항상 새로운 하위 폴더를 생성**한다.
- 생성되는 파일은 모두 해당 하위 폴더 내부에 저장한다.

## 실행 환경

- GPU, CUDA, Docker, NumPy, PyTorch, TorchVision, OpenCV 등 실행 환경이 필요한 분석, 코드 작성, 검증, 추론 및 실험은 모두 **Docker 컨테이너 `HPEBERT_JYSUH` 내부에서 수행**한다.
- 실행 전 반드시 **프롬프트가 실행된 프로젝트의 루트 디렉터리로 이동(`cd`)한 후** 작업을 수행한다.
- 프로젝트 외부 디렉터리에서 실행하지 않는다.
- 실행 환경 구축을 위해 원본 프로젝트를 수정하지 않는다.

## 작업 원칙

- 기존 프로젝트의 구조와 파일은 변경하지 않는다.
- 기존 프로젝트는 분석 및 실행 대상으로만 사용한다.
- 모든 작업은 원본 프로젝트와 독립적으로 수행되어야 하며, 생성되는 모든 산출물은 `Codex_report` 디렉터리에서만 관리한다.
